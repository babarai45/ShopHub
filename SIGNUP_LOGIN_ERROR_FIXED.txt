IMPORTANT NOTE:

The issue was that Django processes template tags BEFORE checking if they're
in HTML comments. Even though the code was in HTML comments (<!-- -->),
Django still tried to evaluate {% provider_login_url 'google' %}, which
failed because:

1. The allauth package was installed
2. But Google OAuth app wasn't configured in admin
3. So the template tag raised a DoesNotExist error

The fix was to completely remove this problematic code rather than trying
to comment it out, since comments don't prevent template tag evaluation.
