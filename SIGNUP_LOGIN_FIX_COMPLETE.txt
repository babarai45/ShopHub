THE PROBLEM
═══════════════════════════════════════════════════════════════════════════

Error: DoesNotExist at /signup/ and /login/

What happened:
  1. User tried to access signup page
  2. Django raised DoesNotExist error
  3. Error was in template rendering
  4. Specifically: {% provider_login_url 'google' %} template tag
  5. This tag tried to fetch Google OAuth app from database
  6. App wasn't configured, so it raised DoesNotExist error

Root cause:
  Template tags are evaluated BEFORE HTML comment processing.
  Even though the code was in HTML comments, Django still tried to process it.

═══════════════════════════════════════════════════════════════════════════
THE SOLUTION
═══════════════════════════════════════════════════════════════════════════

Removed the problematic section from signup.html:
  ❌ BEFORE: Had HTML comment with {% provider_login_url 'google' %}
  ✅ AFTER:  Completely removed that section

Removed the problematic section from login.html:
  ❌ BEFORE: Had HTML comment with {% provider_login_url 'google' %}
  ✅ AFTER:  Completely removed that section

What was removed:
  - "Or sign up with" / "Or continue with" divider sections
  - All Google OAuth SVG icons
  - All template tags that referenced provider_login_url
  - All HTML comments containing the problematic code

What was kept:
  - Simple navigation links ("Sign In" / "Sign Up")
  - All form validation
  - All functionality except Google OAuth option
